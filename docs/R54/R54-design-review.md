## 关键决策

| # | 决策项 | 方案 | 理由 |
|:-:|:-------|:-----|:------|
| D1 | 退回状态存储 | `rejected_steps` 存入 `_PIPELINE_STATE` | 与管线 state 共存，一致性保证 |
| D2 | 退回后 task 处理 | 原 task 标 `INPUT_REQUIRED` + 创建新 task | 保留审计线索，复用现有状态 |
| D3 | git 验证方式 | `git ls-remote` HTTP 协议 | 无需 API key，纯 stdlib |
| D4 | 序列化策略 | 2 秒内存缓冲 | 零 DB 开销，零锁复杂度 |
