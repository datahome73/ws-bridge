## 3. 🅱️ 诊断：F-9 Web 端 Tab 空白（~30min）

### 3.1 已知信息

- **现象：** Web 端 Tab 页加载后持续显示「加载中」，内容区空白
- **历史：** 最早记录于 TODO F-9，标注 🔴 P0，从未排期修复
- **相关代码：** `server/web_viewer.py` + `server/templates.py` + WebSocket 连接状态

### 3.2 诊断方法

| 步骤 | 操作 | 预期产出 |
|:-----|:-----|:---------|
| ① | 检查 Web 容器进程存活 | `ps aux | grep web` / docker ps |
| ② | 检查 Web 端口监听 | `ss -tlnp | grep <port>` |
| ③ | curl 测试 HTTP 响应 | HTTP 状态码 + body 片段 |
| ④ | 检查 server.log / nginx log | 错误堆栈 / 连接拒绝 |
| ⑤ | 浏览器 DevTools Network Tab | JS/CSS/API 调用失败详情 |
| ⑥ | Gateway 连接状态检查 | Gateway 实例数 + 在线状态 |
| ⑦ | WebSocket 握手日志 | `ws://` 连接请求是否 101 |

### 3.3 诊断产出

- **根因定位：** 如「Web 容器 OOM」「Gateway 连接断开」「API 路由 404」等精准结论
- **修复预估：** 小（配置改动 <5min）/ 中（代码改动 <30min）/ 大（架构改动 >1h）
- **影响面：** 只影响 Web 端 or 影响全部服务
- **建议：** 本轮顺手修 vs 留 R71

### 3.4 顺手修复条件

仅在以下条件**全部**满足时在本轮修复：

1. 根因是**配置/部署问题**（非代码架构改造）
2. 修复改动 **≤10 行** 或 重启容器即可
3. 修复**不影响**验证轮核心目标（不引入新风险）
