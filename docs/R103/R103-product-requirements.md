# R103: Web UI 工作区面板增强 — 管线轮次与成员信息显示

> **版本：** v1.0
> **日期：** 2026-07-13
> **状态：** 📝 需求文档
> **轮次：** R103
> **归属管线：** ws-bridge 开发

---

## 一、背景

当前 Web UI 右上角「📋 历史工作室」面板列出所有工作区时，只显示名字和状态（活跃/归档）。但实际工作区的 `Workspace` 数据模型中已有 `pipeline_round`（如 "R102"）、`roles`（所需角色列表）等字段。

PM 在工作区列表中无法一眼看出：
- 这个工作区属于哪个管线轮次？
- 这个工作区有多少成员？
- 这个工作区是哪个轮次的？

→ 需要将已有数据展示到 UI 上。

## 二、需求

### 2.1 后端 — workspace API 补全字段

**文件：** `server/workspace_api.py` 的 `api_workspaces()` 函数

在返回的 workspace dict 中增加：

| 字段 | 类型 | 来源 | 说明 |
|:-----|:-----|:------|:------|
| `pipeline_round` | str | `w.pipeline_round` | 如 "R102"，空字符串表示未关联管线 |
| `roles` | list[str] | `w.roles` | 管线所需角色列表，如 `["pm", "arch", "dev"]` |
| `member_count` | int | 已存在 | 已存在，无需改动 |
| `created_at` | float | 已存在 | 已存在，无需改动 |

改动量：+2 字段追加，约 2 行。

### 2.2 前端 — 工作区面板显示优化

**文件：** `server/templates.py` 的 `buildWsItem()` 函数和 HTML 样式

在现有工作区面板条目中增加：

| 新增元素 | 位置 | 样式 |
|:---------|:------|:------|
| 🏷️ 管线轮次标签 | 名字后面（右侧） | 小号圆角标签，活跃=绿色边框，归档=灰色 |
| 👥 成员计数 | 轮次标签右侧 | 小号文字，如 `成员:3` |
| 📅 创建时间 | 最右侧（归档工作区） | 已有 `closed_at` 显示，增加 `created_at` |

### 2.3 不需要改动的

| 项目 | 原因 |
|:-----|:------|
| Server 消息路由（`main.py`/`handler.py`） | 纯前端/API 增强，不涉及消息处理 |
| 认证/权限系统 | 无关 |
| WS 连接/消息协议 | 无关 |
| Inbox / _inbox:server 中继 | R102 体系不动 |
| Web 服务启动流程 | 不涉及 |

## 三、验收标准

| # | 验收项 | 验证方法 |
|:-:|:-------|:---------|
| 1 | `/api/workspaces` 返回 `pipeline_round` 字段 | `curl /api/workspaces \| jq '.workspaces[0].pipeline_round'` |
| 2 | `/api/workspaces` 返回 `roles` 字段 | `curl /api/workspaces \| jq '.workspaces[0].roles'` |
| 3 | 活跃工作区面板项显示 🏷️ 轮次标签 | 打开 Web 端工作区面板目视确认 |
| 4 | 归档工作区面板项显示轮次 + 创建/关闭时间 | 打开面板查看归档区 |
| 5 | 无轮次关联的工作区不显示错误（空轮次不显示标签） | 新建测试工作区，确认面板显示正常 |
| 6 | CSS 样式兼容手机端 | 调整页面宽度测试响应式 |
| 7 | 不影响现有工作区列表功能（排序、点击切换、缓存） | 确认面板打开/关闭/点击切换正常 |
| 8 | 轮次标签使用统一颜色体系 | 活跃=🟢/归档=灰色，与已有状态颜色一致 |

## 五、风险与注意事项

| 风险 | 等级 | 缓解 |
|:-----|:-----|:------|
| 旧工作区的 `pipeline_round` 为空字符串 | 🟢 无 | 前端判断空字符串时不显示标签 |
| 新 JSON 字段不影响旧客户端 | 🟢 无 | API 响应增加了字段，旧 JS 读取时忽略未知字段 |
| Docker 重建后需清浏览器缓存 | 🟡 注意 | 部署后通知用户 Ctrl+F5 刷新 |
