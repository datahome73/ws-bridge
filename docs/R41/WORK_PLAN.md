**验证方案需覆盖以下四个方向：**

| 方向 | 内容 | 验证目标 | 可能需要的修复 |
|:----:|:-----|:---------|:-------------:|

**验证方案结构：**

```
Part A — 验证（必做）
├── A-1 ~ A-N：每个方向列出验证项目
├── 每个验证项标注：验证方法、预期行为、当前实际行为
└── 结论：⏳ 待验证 → 🟢 正常 / 🔴 需修复

Part B — 修复方案（条件性，仅验证发现异常时产出）
├── 每个需修复的方向给出具体改动建议
├── 改动文件、行号范围、预期 diff
└── Scope 确认：不跨类，不影响已验证的正常功能
```

**验证方案在工作室讨论通过后** → 进入下一 Step。

1. **本轮不做新功能** — 所有改动必须是 Bug 修复或已有功能稳定化
