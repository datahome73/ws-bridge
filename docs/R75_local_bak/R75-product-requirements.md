#### A1 — 内部角色名替换规则

**替换映射表：**

|:------|:-----------|:------|

**替换范围：**

- **不替换：** 历史代码、server/ 目录、config/ 等生产代码（不在本轮范围）

**替换后格式示例：**

```markdown

| 🧐 PM | 小谷 | 需求调研 → 产出需求文档 |
| 🦸 Admin | 小爱 | 创建/关闭工作室 + 合并发布 |

# 改造后
| 🧐 PM | 需求分析师 | 需求调研 → 产出需求文档 |
| 🦸 Admin | 项目管理 | 创建/关闭工作室 + 合并发布 |
```

#### A2 — agent_id 清理

- 替换 `ws_[0-9a-f]{12}` 格式的 agent_id 为通用占位符 `<agent_id>`
- 仅清理文档中的 agent_id，不清理代码中的

#### A3 — 脱敏验证脚本

新增 `scripts/desensitize-check.sh` 用于自动化检查：

```bash
#!/bin/bash
# 检查 docs/ 下所有 .md 文件是否含内部名
PATTERNS="小谷|小爱|小开|爱泰|小周|泰虾|大宏|agent_id: ws_"
FAIL=0
for f in $(find docs/ -name '*.md' -not -path '*/R75/*'); do
    if grep -qE "$PATTERNS" "$f"; then
        echo "❌ 残留: $f"
        FAIL=1
    fi
done
[ "$FAIL" -eq 0 ] && echo "✅ 脱敏验证通过" || exit 1
```

脚本位置：`scripts/desensitize-check.sh`

### 5.1 改动估算

| 文件 | 改动类型 | 估算 |
|:-----|:---------|:-----|
| `docs/R{NN}/WORK_PLAN.md` × 43 | 内容替换（内部名→通用角色名） | ~替换 80 处 |
| `docs/README.md` | 修改（最新轮次更新 + 脱敏） | ~5 行 |

### 5.2 关键风险

| 风险 | 影响 | 缓解 |
|:-----|:------|:------|
| 角色名替换误伤代码或文档中的自然语言 | 替换后内容可能变奇怪 | 使用精确匹配（`小谷` 替换为 `需求分析师`，非正则模糊替换） |
| 提交后新的内部名又通过新轮次涌入 | 脱敏成果被覆盖 | 提交 `scripts/desensitize-check.sh` 作为 pre-commit 检查参考 |
| 脱敏后角色映射表变了，旧轮次文档的可读性降低 | 新读者看不懂旧 MVP 评审记录 | 首轮用 `replace: '小谷' → '需求分析师'` 的可读替换，保留轮次上下文中「PM」的语义 |
