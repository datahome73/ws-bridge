## 方向 A — 流水线单次触发优化（R35-1）

### 背景与痛点

当前启动一轮开发需要多次人工干预：

```
前置决策通过 ✅
    │
    ├─① 项目负责人转发 code 给小爱：『建工作室』
    │
    ├─② 小爱建 ws:R{N}-dev 工作室
    │
    ├─③ 项目负责人再转发 code 给泰虾：『点名报道』
    │
    ├─④ 泰虾主持点名 → 全员确认
    │
    └─⑤ 管线全自动跑完（方案→编码→审查→测试→合并）
         ↑ 到这里才进自动化
```

**痛点：**
1. 项目负责人需要手工转发两次（建工作室 + 点名），属于不必要的机械操作
2. 建工作室和点名之间的等待时间可以消除
3. 当前只有管理员（小爱）有创建工作室的权限，其他角色无法独立触发

### 目标流程

```
前置决策通过 ✅
    │
    ├─① 项目负责人发一次指令（或需求分析师/调度员触发）
    │
    ├─② 自动建工作室 + 自动发送点名通知
    │    ├─ 工作室自动创建（含预定义成员列表）
    │    └─ 自动发送「点名开始」消息到工作室
    │
    ├─③ 泰虾执行点名（流程不变：3分钟超时，回复到齐）
    │    └─ 点名中强制活跃频道确认（见方向 B）
    │
    └─④ → 方案→编码→审查→测试→合并（全自动，不变）
```

### 验收标准

| # | 验收标准 | 优先级 |
|:-:|:---------|:------:|
| A-1 | 前置决策全部通过后，可**一次触发**完成「建工作室 + 发点名通知」两步 | 🔴 P0 |
| A-2 | 触发者不限于管理员，需求分析师/调度员在自己权限范围内可触发建工作室 | 🟡 P2 |
| A-3 | 工作室自动创建时成员列表来自 WORK_PLAN 中的角色分工 | 🟡 P2 |
| A-4 | 点名通知自动发到新建的工作室，通知中包含标准点名文案 | 🟡 P2 |

## 方向 B — 消息污染大厅治理（R36-1）

### 背景与痛点

R36 开发轮中发现：点名报道完成后，部分成员的消息落入大厅而非工作室。

```
点名报道
    │
    ├─ 泰虾：「点名！@全员 请回复到」
    ├─ 小爱：「到」
    ├─ 小开：「到」
    ├─ 泰虾：「到齐，开始工作」
    │
    └─ ❌ 但部分成员的活跃频道还在大厅
         └─ 讨论消息落入大厅，污染公共资源
```

**根因：** 点名报道的实质操作是 `MSG_SET_ACTIVE_CHANNEL` 切换，但当前流程中「回复到」仅表示在线，**不意味着频道已切**。部分成员在执行点名报道前活跃频道可能在其他工作室/大厅，点完名后频道未跟随更新。

**已有基础：** WORKSPACE_RULES.md §26-28 已写入规则，但点名流程中缺少强制执行步骤。

### 目标变化

```
点名报道
    │
    ├─① 泰虾：「点名！@全员 请回复到」
    │     ├─ 全员回复「到」
    │     └─ 服务端自动发送 MSG_SET_ACTIVE_CHANNEL → R37-dev
    │
    ├─② 泰虾：「活跃频道已切换，请回复『已切』确认」
    │     └─ 全员回复「已切」
    │
    ├─③ 服务端：
    │     ├─ 验证所有成员活跃频道 == R37-dev
    │     └─ 通知：点名完成 ✅
    │
    └─④ 开始工作（工作室消息不会再漏到大厅）
```

### 验收标准

| # | 验收标准 | 优先级 |
|:-:|:---------|:------:|
| B-1 | 点名流程中，泰虾发起点名后，服务端自动向全员发送 `MSG_SET_ACTIVE_CHANNEL` 切换到当前工作室 | 🔴 P0 |
| B-2 | 点名主持人在收到全员「到」后，服务器自动验证所有已到成员的活跃频道是否正确切换 | 🔴 P0 |
| B-3 | 未成功切换频道的成员，点名主持收到提示（可单独处理） | 🟡 P2 |
| B-4 | 点名完成后，全员活跃频道锁定在工作室，默认消息发到工作室而非大厅 | 🔴 P0 |

## 当前流程（v5.2）需要调整的部分

### 调整前

```
🟢 Step 1  建工作室        ← 项目管理（小爱）手动执行
🟢 Step 2  点名报道         ← 测试工程师（泰虾）手动主持
                            ├─ 无强制频道切换验证
```

### 调整后

```
🟢 Step 1  建工作室        ← 自动触发（需求分析师/项目管理均可启动）
🟢 Step 2  点名报道 + 频道锁定  ← 测试工程师主持
                            ├─ 服务端自动 MSG_SET_ACTIVE_CHANNEL
                            ├─ 全员回复「已切」确认
                            └─ 服务端验证完成后通知点名完成 ✅
```

### WORKFLOW.md 变更

| 变更点 | 旧 | 新 |
|:-------|:---|:---|
| Step 1 触发方式 | 项目管理手动建 | 可通过 `!` 触发词自动建工作室 |
| Step 2 频道切换 | 隐式，不强制执行 | 显式 MSG_SET_ACTIVE_CHANNEL + 全员确认 |
| Step 2 完成条件 | 全员回复「到」 | 全员回复「到」+ 全员回复「已切」+ 服务端验证通过 |
