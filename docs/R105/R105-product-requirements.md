# R105: 统一 Bot 回复模板 — 修复 ACK/完成/退回/失败格式

> **版本：** v1.0
> **日期：** 2026-07-13
> **状态：** 📝 需求文档
> **轮次：** R105

---

## 一、背景

R102 部署了 Server 中继体系（`_inbox:server` + 前缀匹配），但各 bot 在回复时频繁出现格式错误：

| Bot | R103 问题 | R104 问题 |
|:----|:----------|:----------|
| 小开 | ✅ 放末尾，前缀不命中 | ✅ 放末尾，前缀不命中 |
| 爱泰 | 发到 `_system` + 无前缀 | — |
| 小周 | ✅ 正确 | ✅ 正确 |
| 泰虾 | 无 ACK + `✅ ... 完成` 而非 `已完成 ✅` | — |
| 小爱 | 无 ACK | — |

**Server 端已就绪**（前缀匹配规则已就位），问题是各 bot 的客户端不知道正确的回复格式。需要统一配置。

## 二、需求

### 2.1 标准回复模板

所有 bot 统一使用以下 4 个场景的回复格式：

| 场景 | 前缀格式 | 频道 | 说明 |
|:-----|:---------|:------|:------|
| **ACK（收到任务）** | `收到 ✅ R{N} Step {N}` | `_inbox:server` | 收到派活后立即回复 |
| **完成** | `已完成 ✅ R{N} Step {N}，已推 dev: {sha}` | `_inbox:server` | 完成工作后回复 |
| **退回** | `退回 🔄 R{N} Step {N} — {原因}` | `_inbox:server` | 无法完成，退回 PM |
| **失败** | `失败 ❌ R{N} Step {N} — {原因}` | `_inbox:server` | 执行失败，需要协助 |

**重要：**
- `✅` / `🔄` / `❌` 符号**必须放在消息开头**（Server 前缀匹配只检查开头）
- 所有回复统一发到 **`_inbox:server`**（不是 `_system`）
- `R{N}` 和 `Step {N}` 由实际轮次和步骤号替换

### 2.2 Server 端前缀匹配规则（已存在，无需修改）

```
收到 ✅ → ACK，转发 PM + 自动回复 bot
已完成 ✅ → 完成通知，转发 PM + 自动回复 bot
退回 🔄 → 退回通知，转发 PM + 自动回复 bot
失败 ❌ → 失败通知，转发 PM + 自动回复 bot
✅      → 通用前缀，转发 PM（兜底）
```

### 2.3 修改方式

**方案：创建回复格式 Skill（集中文档模式）**

参考 `skills/` 目录下的入驻 skill（`ws-bridge-registration`），在 ws-bridge 仓库 `skills/` 目录下创建一份回复格式协议文档，各 bot 自行阅读并遵守。只需在 bot 提示词中加一行引用即可。

**① 新建 `skills/reply-format-protocol.md`**

集中定义 4 种场景的回复格式：

```markdown
# 回复格式协议

> ws-bridge bot 回复格式规范。所有 bot 在回复 inbox 任务时必须遵守。

---

## 四种回复场景

收到 PM 通过 inbox 派发的任务后，根据状态回复到 `_inbox:server`：

| 场景 | 前缀格式 | 说明 |
|:-----|:---------|:------|
| 📩 ACK | `收到 ✅ R{轮次} Step {步骤}` | 收到任务后立即回复 |
| ✅ 完成 | `已完成 ✅ R{轮次} Step {步骤}，已推 dev: {sha}` | 完成任务后回复 |
| 🔄 退回 | `退回 🔄 R{轮次} Step {步骤} — {原因}` | 无法完成时退回 |
| ❌ 失败 | `失败 ❌ R{轮次} Step {步骤} — {原因}` | 执行失败 |

## 核心规则

1. **频道固定为 `_inbox:server`** — 禁止发到 `_system`
2. **`✅`/`🔄`/`❌` 必须放在消息开头** — Server 前缀匹配只看开头
3. `R{N}` 和 `Step {N}` 替换为实际轮次和步骤号
4. 每条回复 ≤200 字，不要拆多条，不要寒暄

## 示例

```
收到 ✅ R105 Step 3
已完成 ✅ R105 Step 3，已推 dev: abc1234
退回 🔄 R105 Step 3 — 缺少前置依赖文档
失败 ❌ R105 Step 3 — 编译错误，需运维协助
```
```

**② 各 bot 提示词加一行引用**

```
回复格式请参考 skills/reply-format-protocol.md（_inbox:server，✅/🔄/❌ 放开头）
```

**优势：**
- 集中维护，一处更新所有 bot 生效
- 与入驻 skill 同样模式，bot 已熟悉
- 文档本身就是 protocol 规范

## 五、风险与注意事项

| 风险 | 等级 | 缓解 |
|:-----|:-----|:------|
| Bot 配置更新后不生效（缓存/记忆残留） | 🟡 | 配置更新后建议重启/重建 bot 连接 |
| Bot 回复仍走 `_system` 而非 `_inbox:server` | 🟡 | 在提示词中强调频道字段 |
